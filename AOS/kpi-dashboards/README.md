## Structure

Every employee dashboard follows the same shape, so the CEO Dashboard
can roll them up consistently:

- **Inputs** — what the employee consumes
- **Outputs** — what it produces
- **Success Metrics** — what's actually measured, from the employee's
  own operating manual
- **Weekly Targets** — a starting bar, not a fixed law
- **Monthly Targets** — the same, at a longer horizon
- **Business Impact** — why this employee's number matters to the
  practice as a whole, not just to itself

## On the Targets

The weekly/monthly numbers below are starting targets, not historical
performance — every JSON log in AOS starts empty by design (see the
root `README.md`), so there is no real data yet to derive targets from.
Treat them as a first calibration to revise once a few real cycles of
`daily-operating-workflow.md` have run, not as commitments already
being tracked against actuals.
